一、开发分支（dev）上的代码达到上线的标准后，要合并到 master 分支
git checkout dev
git pull
git checkout master
git merge dev
git push -u origin master

二、当master代码改动了，需要更新开发分支（dev）上的代码
git checkout master 
git pull 
git checkout dev
git merge master



Git更新本地冲突：commit your changes or stash them before you can merge。。。
方法一：stash
git stash
git pull
git stash pop

方法二：直接完全覆盖本地修改
git reset --hard
git pull

回退到merge前
git reset --merge
